# Public Launch Checklist

## Before Publish

- Create a public GitHub repository.
- Set repository description: `Evidence-bound maintainer packets and Codex workflow plans for OSS projects.`
- Add topics: `open-source`, `maintainer-tools`, `codex`, `github`, `release-workflows`.
- Push the initial commit.
- Confirm CI passes.
- Create `v0.1.0` release with the changelog text.

## First Issues To Open

- Add GitHub API enrichment for open issues and pull requests.
- Add package registry signal collectors for npm and PyPI.
- Add `release-packet` output mode.
- Add examples from 3 public repositories with maintainer permission.
- Add JSON output for automation.

## First Outreach

- Ask maintainers to try the tool on their own public repo.
- Ask for issue comments with missing evidence sources.
- Do not ask for artificial star swaps.
- Publish one concrete dogfood example and one limitation note.
