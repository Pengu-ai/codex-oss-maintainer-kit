# Security Policy

## Reporting

Please report security issues privately to the project maintainer before opening a public issue.

## Scope

This project reads local Git metadata and writes Markdown packets. It should not require repository secrets, API keys, or credentials for the default workflow.

## Safety Rules

- Do not paste private repository data into public examples.
- Do not include API keys, tokens, cookies, or private maintainer notes in generated packets.
- Review generated Markdown before publishing it.
